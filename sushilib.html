SushiLib

Thư viện (Core Plugin) xương sống của toàn bộ Hệ sinh thái Sushi.

Tải Xuống Ngay
Phiên bản: 1.20+
Yêu cầu (Dependencies): Vault, PlaceholderAPI

Mô Tả

SushiLib v3.0

SushiLib là thư viện (Core Plugin) xương sống của toàn bộ Hệ sinh thái Sushi. Plugin này không chỉ đóng vai trò là một thư viện cung cấp API cho lập trình viên, mà còn sở hữu một hệ thống Chỉ số (RPG Stats), Hiệu ứng (Aura), Phòng thủ/Sát thương và Quản lý tiền tệ mạnh mẽ cho Server Minecraft.

🌟 Tính Năng Nổi Bật

  • Hệ thống Chỉ số RPG Cốt lõi (Stats API):
    • Quản lý máu, năng lượng, thể lực, tốc độ di chuyển, sát thương, chí mạng, đỡ đòn, v.v.
    • Hỗ trợ công thức tính toán tuỳ chỉnh hoàn toàn thông qua file stats.yml (Sử dụng biểu thức toán học thay vì hard-code).
    • Hệ thống giới hạn chỉ số (min-max-values), quy định base-stat (base-stat-value) và định dạng số thập phân (decimal-format).
  • Hệ thống Nguyên Tố (Element System): Hệ thống cộng hưởng sát thương hệ Lửa, Nước, Đất, Gió thông qua file elements.yml.
  • Hệ thống Combat (DamageManager): Tính toán sát thương tự động với xuyên giáp, kháng hệ, phòng thủ, lifesteal (hút máu) và phản đòn.
  • Hệ thống Hiệu ứng Tạm thời (Aura & Temp Stat): Thêm chỉ số hoặc hiệu ứng tạm thời trong thời gian nhất định (Hỗ trợ qua lệnh Admin).
  • Hệ Thống Tiền Tệ & Hồi Chiêu (Economy & Cooldown): Tích hợp sâu vào Vault. Cung cấp API quản lý hồi chiêu thời gian thực.
  • Tối ưu Cơ Sở Dữ Liệu: Sử dụng SQLite local hoặc MySQL (HikariCP) để lưu trữ chỉ số. Đồng bộ đa luồng.
  • Tương thích PlaceholderAPI: Cung cấp hàng loạt biến số cho Holograms, Scoreboard.

📝 Danh Sách Lệnh (Commands)

Alias khả dụng: /sushilib hoặc /sl

Lệnh Quyền hạn (Permission) Chức năng
/sl help Tất cả Hiển thị danh sách lệnh.
/sl attributes Tất cả Mở bảng cộng điểm tiềm năng.
/sl points give <player> <amount> sushilib.command.points Phát điểm tiềm năng.
/sl aura give <player> <type> <seconds> sushilib.admin Ép hiệu ứng Aura.
/eco <give/take/set> <player> <currency> <amount> sushilib.command.eco Quản lý tiền tệ.

⚙️ Hệ Thống Cấu Hình (Configuration Files)

Khi tải plugin lần đầu, 3 file chính sẽ được tạo trong thư mục plugins/SushiLib/:

1. config.yml

Quản lý các kết nối Database, cấu hình cơ bản, Placeholder hiển thị Holograms.

2. stats.yml (Quan trọng nhất)

File quyết định toàn bộ sức mạnh và cân bằng (Balance) của server.

  • decimal-format: Quy định cách làm tròn số trong Menu/GUI (VD: CRITICAL_CHANCE: '0.##').
  • min-max-values: Khống chế sức mạnh tuyệt đối, ngăn chặn lạm phát.
  • base-stat-value: Chỉ số cơ bản mặc định.
  • formulas: Nơi định nghĩa công thức tính toán toán học sát thương.

🛠️ Dành cho Lập Trình Viên (For Developers)

Nếu bạn code một plugin Add-on cho Sushi, bạn có thể gọi API thông qua:

import com.sushilib.SushiLib;

// Lấy máu tối đa của người chơi
double maxHP = SushiLib.getAPI().getStats().getStat(player, "MAX_HEALTH");

// Thêm buff cho người chơi trong 1 khoảng thời gian (StatModifier)
StatModifier buff = new StatModifier("BUFF_SUC_MANH", 50.0, ModifierType.FLAT);
SushiLib.getAPI().getStats().addModifier(player, "WEAPON_DAMAGE", buff);
