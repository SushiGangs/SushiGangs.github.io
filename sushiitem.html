SushiItems

Hệ thống vật phẩm RPG với thuộc tính tùy chỉnh và tối ưu hóa cực cao thông qua PDC.

Tải Xuống
Phiên bản:1.20+
Yêu cầu:SushiLib

Mô Tả

1. Tổng Quan Custom Items

SushiItems là một framework hỗ trợ thiết kế, quản lý và sử dụng Custom Items mạnh mẽ cho Server Minecraft, hướng tới việc tối ưu hóa hiệu suất (NBT/PDC) và khả năng tùy chỉnh linh hoạt thông qua cấu hình YAML.

  • Quản lý Item bằng hệ thống ID Độc Nhất (NamespacedKey).
  • Tương thích hoàn hảo với tài nguyên gói hình ảnh (Custom Model Data / Oraxen / ItemsAdder).

2. Cấu trúc YAML Item Cơ Bản

Sử dụng YAML để định nghĩa vật phẩm, tích hợp trực tiếp với hệ thống chỉ số của SushiLib.

# Mẫu vật phẩm: Thanh Gươm Của Rồng
dragon_sword:
  material: DIAMOND_SWORD
  custom_model_data: 1001
  display_name: "&c&lThanh Gươm Của Rồng"
  lore:
    - "&7Thanh gươm được rèn từ lửa rồng."
    - ""
    - "&c✦ Sát thương: +50"
    - "&6✦ Chí mạng: +15%"
  rpg_stats:
    DAMAGE: 50
    CRIT_CHANCE: 15
  requirements:
    level: 20
    class: "warrior"

3. Tối ưu hóa Dữ Liệu (PDC)

SushiItems không sử dụng NBT (Named Binary Tag) truyền thống vốn gây nặng Server. Thay vào đó, toàn bộ dữ liệu Item (ID, Stats, Custom Flags) được lưu trữ qua PersistentDataContainer (PDC) - API gốc của Spigot/Paper, mang lại hiệu năng cao nhất và chống copy/hack nbt.

4. API cho Developer

// Lấy dữ liệu Custom Item từ ItemStack
ItemStack item = player.getInventory().getItemInMainHand();
SushiItem sItem = SushiItems.getAPI().getSushiItem(item);

if (sItem != null) {
    String id = sItem.getId();
    double dmg = sItem.getStat(StatType.DAMAGE);
}

// Chế tạo Item mới
ItemStack generatedItem = SushiItems.getAPI().buildItem("dragon_sword");

5. Lệnh & Permissions

  • /sushiitem get <id> [amount] - Lấy vật phẩm. (sushiitems.admin)
  • /sushiitem give <player> <id> [amount] - Phát vật phẩm cho người chơi.
  • /sushiitem list - Mở GUI danh sách toàn bộ vật phẩm.
  • /sushiitem update - Cập nhật lại chỉ số của vật phẩm đang cầm trên tay theo config mới nhất.
