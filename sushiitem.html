SushiItems

Hệ thống tạo và quản lý vật phẩm tùy chỉnh (Custom Items) vô cùng mạnh mẽ.

v1.0.0 Yêu cầu: SushiLib, SushiCore

Mô Tả

Tải Xuống Ngay

1. Tổng Quan Custom Items

SushiItems là một framework hỗ trợ thiết kế, quản lý và sử dụng Custom Items mạnh mẽ cho Server Minecraft, hướng tới việc tối ưu hóa hiệu suất (NBT/PDC) và khả năng tùy chỉnh linh hoạt thông qua cấu hình YAML.

  • Persistent Data Container (PDC): Gắn dữ liệu trực tiếp vào NBT tag ẩn của item, không bị mất khi giao dịch hay rớt đồ.
  • Custom Model Data: Hỗ trợ gọi các resource pack tùy chỉnh 2D, 3D thông qua NBT ModelData.
  • Stat Injector: Nhúng (inject) trực tiếp các chỉ số RPG (Damage, Health, Defense) từ SushiLib vào vật phẩm.
  • Skill Binding: Trói buộc một kỹ năng cụ thể từ SushiSkills vào vật phẩm.

2. Cấu Trúc File Vật Phẩm

Dễ dàng tạo vũ khí, áo giáp, nguyên liệu bằng YAML:

items:
  dragon_sword:
    material: DIAMOND_SWORD
    model_data: 1024
    name: "&c&lKiếm Hỏa Long"
    lore:
      - "&7Thanh kiếm rèn từ dung nham."
      - ""
      - "&c⚔ Sát Thương Lửa: 250"
    
    # Kết nối với SushiLib Stats
    rpg_stats:
      PHYSICAL_DAMAGE: 150
      FIRE_DAMAGE: 100
      CRIT_CHANCE: 15
    
    # Kết nối kỹ năng (SushiSkills)
    bound_skill: "dragon_breath"

3. Dành Cho Dev (API)

Bạn có thể lấy hoặc tạo Item thông qua API của hệ thống:

import com.sushiitems.api.SushiItemAPI;

// Lấy vật phẩm theo ID
ItemStack dragonSword = SushiItemAPI.getItem("dragon_sword");

// Kiểm tra xem item trên tay người chơi có phải custom item không
if (SushiItemAPI.isCustomItem(itemStack)) {
    String itemId = SushiItemAPI.getItemId(itemStack);
    // ... xử lý logic
}

4. Lệnh Hệ Thống

  • /si give <player> <item_id> [amount]: Cấp vật phẩm trực tiếp (Hỗ trợ gọi từ Console cho các plugin Vote/Donate).
  • /si browse: Mở Menu UI xem toàn bộ danh sách vật phẩm đang có trên Server.
  • /si reload: Quét lại toàn bộ thư mục cấu hình và cập nhật vật phẩm lập tức mà không cần khởi động lại.
